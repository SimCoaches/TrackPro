# EmailJS Setup Guide for TrackPro Support

This guide explains how to set up EmailJS to enable automatic email delivery for support tickets submitted through TrackPro.

## Overview

EmailJS allows TrackPro to send support tickets directly to your support email without needing a backend server. Users can submit support tickets through the TrackPro interface, and they'll be automatically forwarded to your designated support email address.

## Step 1: Create EmailJS Account

1. Go to [EmailJS.com](https://www.emailjs.com/)
2. Sign up for a free account (allows 200 emails/month)
3. Verify your email address

## Step 2: Set Up Email Service

1. In your EmailJS dashboard, go to **Email Services**
2. Click **Add New Service**
3. Choose your email provider (Gmail, Outlook, etc.)
4. Follow the setup instructions for your provider:
   - **Gmail**: Use OAuth2 authentication
   - **Outlook**: Use OAuth2 authentication
   - **Custom SMTP**: Enter your SMTP settings

## Step 3: Create Email Template

1. Go to **Email Templates** in your EmailJS dashboard
2. Click **Create New Template**
3. Use this template content:

```html
Subject: TrackPro Support Ticket - {{ticket_subject}}

New support ticket received from TrackPro:

**Ticket Information:**
- Subject: {{ticket_subject}}
- Priority: {{ticket_priority}}
- Category: {{ticket_category}}
- Submitted: {{timestamp}}

**User Information:**
- Name: {{user_name}}
- Email: {{user_email}}

**Description:**
{{ticket_description}}

---
This ticket was automatically generated by TrackPro Support System.
```

4. Save the template and note the **Template ID**

## Step 4: Get Your Credentials

From your EmailJS dashboard, collect these three pieces of information:

1. **Service ID**: Found in Email Services section
2. **Template ID**: Found in Email Templates section  
3. **Public Key**: Found in Account > API Keys

## Step 5: Configure TrackPro

### Option A: Configuration File (Recommended)

1. Navigate to: `trackpro/ui/pages/support/`
2. Copy `emailjs_config.json.template` to `emailjs_config.json`
3. Edit `emailjs_config.json` with your credentials:

```json
{
  "service_id": "your_actual_service_id",
  "template_id": "your_actual_template_id", 
  "public_key": "your_actual_public_key"
}
```

### Option B: Environment Variables

Set these environment variables:
```bash
EMAILJS_SERVICE_ID=your_actual_service_id
EMAILJS_TEMPLATE_ID=your_actual_template_id
EMAILJS_PUBLIC_KEY=your_actual_public_key
```

## Step 6: Test the Integration

1. Start TrackPro
2. Go to Support page
3. You should see "✅ Email delivery enabled" at the top of the form
4. Submit a test ticket
5. Check your support email to confirm receipt

## Troubleshooting

### Email Not Received
- Check EmailJS dashboard for delivery status
- Verify template variables match the code
- Check spam/junk folder
- Ensure service and template are active

### Configuration Not Detected
- Verify `emailjs_config.json` is in the correct directory
- Check JSON syntax is valid
- Restart TrackPro after configuration changes

### API Errors
- Check your EmailJS account limits
- Verify public key is correct
- Ensure service and template IDs match exactly

## Security Notes

- EmailJS public keys are safe to include in client applications
- Never share your private EmailJS account credentials
- Consider rate limiting if abuse becomes an issue
- Monitor your EmailJS usage dashboard regularly

## Advanced Features

### Custom Template Variables

You can modify the email template to include additional information:
- System information
- TrackPro version
- User preferences
- Attached screenshots (if implemented)

### Multiple Templates

Create different templates for:
- Bug reports
- Feature requests  
- General support
- Critical issues

Then modify the code to use different template IDs based on the ticket category.

## Support

If you encounter issues with this setup:
1. Check the EmailJS documentation
2. Verify all credentials are correct
3. Test with a simple template first
4. Contact EmailJS support for service-specific issues

The fallback system will still work even if EmailJS is not configured - users will see ticket details to manually email to support.
